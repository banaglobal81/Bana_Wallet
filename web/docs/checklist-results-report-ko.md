## 항목 1 — 스테이킹 데일리 이자 (5분 지급)

**테스트 스테이킹:** 5,000 BANA · 10일 상품 · 0.2%/일 · 클럭을 5분 = 1일로 설정
**방법:** 5분마다 실제 정산을 50분간(만기까지) 실행

**공식:** `이자 = 원금 × 이자율 ÷ 100 × 일수 = 5000 × 0.2 ÷ 100 × 일수 = 10 × 일수`

| 실제 경과 | 일수 | 수기 계산 (10 × 일수) | 시스템 지급 | 일치 |
|---|---|---|---|---|
| 0:00 | 0 | 0 | 0 | ✅ |
| 5분 | 1 | 10 | 10 | ✅ |
| 10분 | 2 | 20 | 20 | ✅ |
| 15분 | 3 | 30 | 30 | ✅ |
| 25분 | 5 | 50 | 50 | ✅ |
| 50분 | 10 | 100 | **100 → 만기(MATURED)** | ✅ |

**원장:** 정확히 10건 × 10 BANA = 100 BANA, 이중지급 없음, 정시 만기.
**결과: 통과.** 이자가 5분 간격으로 정확히 지급되며 수기 계산과 일치합니다.

## 항목 2 — MLM 커미션 (대·소실적 매칭 + 유니레벨 부스트)

**테스트 트리 (BANA):**
```
        U  ← 커미션 수령
      / | \
     A  B  C     A = 10,000 @0.5% → 일 50
     |            B =  5,000 @0.2% → 일 10
     A1           C =  5,000 @0.2% → 일 10
                  A1 = 4,000 @0.2% → 일  8
```
**방법:** MLM 엔진 ON + 5분 = 1일; 5분마다 정산을 6일치 실행

### U의 수기 계산 (1일 기준)
**Layer 1 (대·소실적 매칭):**
- 라인: A = 10,000+4,000 = 14,000 (가장 큼 → 제외/대실적) · B = 5,000 · C = 5,000
- 소실적(B+C) 규모 = 10,000 → 레벨 **B2 = 1.5%**
- 소실적 데일리 이자 = 10 + 10 = 20
- Layer 1 = `1.5% × 20` = **0.3**

**Layer 2 (유니레벨 부스트):** 직접 추천 3명 → 1·2·3대 활성화
- 1대(A+B+C) = 70 × 5% = 3.5
- 2대(A1) = 8 × 3% = 0.24
- 3대 = 0
- Layer 2 = **3.74**

**합계 = 0.3 + 3.74 = 일 4.04 BANA**

### 라이브 결과 vs 수기 계산
| 시각 | 일수 | 수기 (4.04 × 일수) | 시스템(U) | 일치 |
|---|---|---|---|---|
| 16:07 | 1 | 4.04 | 4.04 | ✅ |
| 16:12 | 2 | 8.08 | 8.08 | ✅ |
| 16:17 | 3 | 12.12 | 12.12 | ✅ |
| 16:22 | 4 | 16.16 | 16.16 | ✅ |
| 16:27 | 5 | 20.20 | 20.20 | ✅ |
| 16:32 | 6 | 24.24 | 24.24 | ✅ |

- 매일 원장 행: layer1 = 0.3, layer2 = 3.74, 합계 = 4.04.
- 상위 **A**도 자신의 커미션을 정확히 수령(일 0.4 → 2.4) — 모든 단계에서 지급됨.
- 동시에 기본 스테이킹 이자도 정상 적립(A: 일 50 → 5일차 250).
- 멱등성 보장(하루 1행), 플래그 OFF 시 아무것도 지급되지 않음.

**결과: 통과.** 커미션이 자료(덱) 구조대로 지급되며 수기 계산과 일치합니다.

체크리스트 요건대로 항목 1·2에 대해 자동 검증 **및** 수기(계산기) 검증을 모두 수행했으며 둘 다 일치합니다.

## 참고 표 (향후 수치 검증용)

**스테이킹 일일 이자율:** 10일 0.2% · 30일 0.5% · 90일 0.7% · 180일 1.0% · 360일 1.3%(임시값).

**Layer 1 — 소실적 매칭 (기준: BANA):**
| 레벨 | 최소 | % | | 레벨 | 최소 | % |
|---|---|---|---|---|---|---|
| B1 | 3,000 | 1.0 | | B6 | 700,000 | 4.5 |
| B2 | 10,000 | 1.5 | | B7 | 1,500,000 | 5.5 |
| B3 | 30,000 | 2.0 | | B8 | 3,000,000 | 6.6 |
| B4 | 100,000 | 2.8 | | B9 | 7,000,000 | 8.0 |
| B5 | 300,000 | 3.6 | | B10 | 15,000,000 | 10.0 |

**Layer 2 — 단계별 유니레벨 부스트 (1→10대):** 5% · 3% · 3% · 3% · 2% · 2% · 2% · 2% · 3% · 5% (누적 상한 30%). 직접 추천은 해당 계정이 200 BANA 이상 스테이킹 시 인정.

**공식:**
```
이자      = 원금 × 이자율 ÷ 100 × 일수
Layer 1  = (소실적 %) × (소실적 데일리 이자)
Layer 2  = Σ (g대 부스트 %) × (g대 데일리 이자),  g = 1 .. 직접 추천 수
커미션    = Layer 1 + Layer 2   (매일 지급, BANA)
```

## 미결 항목 (코드가 아닌, 프로덕션 출시 전 필요 사항)

1. **유니레벨 기준 확인** — 시니어 확인 필요(현재 가정: 각 단계(대)의 데일리 이자 기준).
2. **법률/규제 검토 승인** — 덱에 DRAFT·별도 검토 필요로 표기됨. 승인 전까지 엔진은 OFF(`REFERRAL_BONUS_ENABLED=false`) 유지.
3. **실제 출금 가능 잔고로 지급 반영** — 현재 이자·커미션은 원장에 기록되나 사용자의 사용 가능 잔고로는 아직 이체되지 않음(둘 다 해당).
4. **데일리 크론 워커 배포** — 프로덕션에서 지급이 매일 자동 실행되도록.
5. **실사용 시나리오 E2E 테스트** — 추천 링크 가입 → 스테이킹 → 이자 → 커미션 → 출금.
